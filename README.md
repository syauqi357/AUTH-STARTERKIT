# 🚀 SvelteKit + Express Starter Kit

A fullstack starter kit with authentication out of the box. Inspired by Laravel Jetstream/Breeze but for the JS ecosystem.

**Stack:** SvelteKit · Express · Better Auth · Drizzle ORM · MySQL

## ⚙️ Prerequisites

- Node.js 18+
- MySQL database (local or remote)
- SMTP credentials (for forgot password email)

---

## 🛠️ Setup

### 1. Clone & Install

```bash
git clone https://github.com/yourusername/your-starter-kit.git
cd your-starter-kit
npm install
```

### 2. Environment Variables

**`apps/backend/.env`**
```env
BETTER_AUTH_SECRET=your_secret_here
BETTER_AUTH_URL=http://localhost:3000
FRONTEND_URL=http://localhost:5173
DATABASE_URL=mysql://user:password@localhost:3306/your_db

# SMTP (for forgot password)
SMTP_HOST=smtp.mailtrap.io
SMTP_PORT=587
SMTP_USER=your_smtp_user
SMTP_PASS=your_smtp_pass
SMTP_FROM=no-reply@yourdomain.com
```

**`apps/frontend/.env`**
```env
PUBLIC_API_URL=http://localhost:3000
```

### 3. Generate Auth Schema

```bash
cd apps/backend
npx better-auth generate
```

This auto-generates `src/db/schema/auth-schema.ts` based on your auth config and plugins.

### 4. Push Schema to Database

```bash
npx drizzle-kit push
```

This syncs your Drizzle schema to MySQL — adds all required tables and columns including `username` and `display_username`.

### 5. Run Development Server

From root:
```bash
npm run dev
```

Or individually:
```bash
npm run dev:frontend   # http://localhost:5173
npm run dev:backend    # http://localhost:3000
```

## 🔐 Auth Features

| Feature | Endpoint |
|---|---|
| Register | `POST /api/auth/sign-up/email` |
| Login (username) | `POST /api/auth/sign-in/username` |
| Logout | `POST /api/auth/sign-out` |
| Forgot Password | `POST /api/auth/forget-password` |
| Reset Password | `POST /api/auth/reset-password` |
| Get Session | `GET /api/auth/get-session` |

## 📄 Pages

| Route | Description | Protected |
|---|---|---|
| `/login` | Login with username + password | ❌ |
| `/register` | Create new account | ❌ |
| `/forgot-password` | Request password reset email | ❌ |
| `/reset-password` | Set new password via token from email | ❌ |
| `/dashboard` | Main app page | ✅ |
| `/profile` | View & edit profile | ✅ |

Protected routes redirect to `/login` if no active session.

## 🔄 Auth Flows

### Register
```
User fills form (name, username, email, password)
  → signUp.email({ name, username, email, password })
  → redirect /dashboard
```

### Login
```
User fills form (username, password)
  → signIn.username({ username, password })
  → redirect /dashboard
```

### Forgot Password
```
User enters email on /forgot-password
  → authClient.forgetPassword({ email, redirectTo: "/reset-password" })
  → Better Auth sends reset email
  → User clicks link → /reset-password?token=xxx
  → authClient.resetPassword({ newPassword, token })
  → redirect /login
```

### Logout
```
User clicks logout
  → authClient.signOut()
  → redirect /login
```

## 📦 Key Dependencies

**Backend**
- `better-auth` — authentication engine
- `@better-auth/drizzle-adapter` — Drizzle adapter for Better Auth
- `drizzle-orm` + `mysql2` — ORM + MySQL driver
- `express` — HTTP server
- `drizzle-kit` — schema generation & migration

**Frontend**
- `better-auth/client` — auth client SDK
- `@sveltejs/kit` — fullstack Svelte framework

---

## 🚢 Adding New Features

1. **New auth plugin** → add to `auth.ts` plugins array → re-run `better-auth generate` → `drizzle-kit push`
2. **New page** → add route in `apps/frontend/src/routes/`
3. **Protected page** → add `+page.server.ts` with session guard
